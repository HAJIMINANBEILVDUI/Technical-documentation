## 1. 引言

### 1.1 系统概述

胸部CT影像肺叶自动分割系统是一个基于nnU-Net深度学习框架的医疗影像分析系统，旨在实现胸部CT影像中肺叶的自动、精确分割。系统通过训练深度学习模型，能够自动识别并分割出左肺上叶、左肺下叶、右肺上叶、右肺中叶和右肺下叶五个肺叶区域，为肺部疾病诊断和治疗提供辅助支持。

### 1.2 目标用户

- 放射科医师
- 医学影像研究人员
- 肺部疾病诊断专家
- 医疗AI研究人员

### 1.3 应用场景

- 肺部疾病诊断辅助
- 肺叶切除手术规划
- 肺部病变定位和定量分析
- 医学影像教学和研究

## 2. 系统范围

### 2.1 包含功能

- 胸部CT影像数据导入和管理
- CT影像预处理（重采样、归一化、裁剪等）
- 基于nnU-Net的肺叶分割模型训练
- 肺叶自动分割推理
- 分割结果可视化展示
- 分割性能评估和指标计算
- 模型版本管理和保存
- 批量处理功能

### 2.2 排除功能

- 病灶检测和分类
- 其他器官分割
- DICOM网络传输功能
- 医疗设备直接对接
- 临床决策支持系统
- 电子病历系统集成

## 3. 功能需求

### 3.1 数据管理模块

#### FR-DATA-001 数据导入功能
**描述**: 系统应支持从本地文件系统导入胸部CT影像数据

**验收标准**:
- 当用户选择胸部CT影像文件时，系统应当在30秒内完成导入
- 当导入的文件格式为NIfTI或DICOM时，系统应当成功解析并加载
- 当导入的影像数据不符合胸部CT规格时，系统应当提示用户并拒绝导入
- 当导入过程中发生错误时，系统应当显示具体的错误信息

#### FR-DATA-002 数据格式支持
**描述**: 系统应当支持多种医学影像格式

**验收标准**:
- 当提供NIfTI格式(.nii, .nii.gz)的CT影像时，系统应当能够正确读取
- 当提供DICOM格式的CT影像时，系统应当能够正确读取
- 当提供其他未支持的格式时，系统应当提示用户支持的格式列表

#### FR-DATA-003 数据集组织
**描述**: 系统应当按照nnU-Net要求的数据集结构组织训练数据

**验收标准**:
- 当用户准备训练数据时，系统应当自动创建nnU-Net标准目录结构
- 当组织数据集时，系统应当将影像和标签文件分别放置在imagesTr和labelsTr目录
- 当组织数据集时，系统应当生成符合nnU-Net要求的dataset.json配置文件

### 3.2 影像预处理模块

#### FR-PRE-001 影像重采样
**描述**: 系统应当对输入的CT影像进行重采样，统一空间分辨率

**验收标准**:
- 当输入CT影像的体素间距与目标间距不同时，系统应当重采样到目标间距
- 当重采样操作执行时，系统应当使用三次样条插值方法
- 当重采样操作完成后，系统应当保持影像的解剖结构完整性

#### FR-PRE-002 影像归一化
**描述**: 系统应当对CT影像的HU值进行归一化处理

**验收标准**:
- 当输入CT影像的HU值范围超出正常肺部组织范围时，系统应当进行裁剪（-1000到400 HU）
- 当归一化操作执行时，系统应当将HU值映射到[0, 1]区间
- 当归一化操作完成后，系统应当保持影像的相对强度关系

#### FR-PRE-003 影像裁剪
**描述**: 系统应当自动裁剪CT影像中的有效肺部区域

**验收标准**:
- 当输入CT影像包含大量背景区域时，系统应当自动裁剪到包含肺部的最小边界框
- 当裁剪操作执行时，系统应当在边界框周围保留至少10像素的边缘
- 当裁剪操作完成后，系统应当记录裁剪参数以便后续恢复原始坐标

#### FR-PRE-004 数据增强
**描述**: 系统应当在训练过程中对影像数据进行增强

**验收标准**:
- 当训练模型时，系统应当对训练数据应用随机旋转、翻转、缩放等增强操作
- 当应用数据增强时，系统应当对影像和标签应用相同的变换
- 当数据增强操作执行时，系统应当确保增强后的数据仍然符合医学影像的合理性

### 3.3 模型训练模块

#### FR-TRAIN-001 nnU-Net配置选择
**描述**: 系统应当支持nnU-Net的不同配置选项

**验收标准**:
- 当用户选择模型配置时，系统应当提供2D、3D lowres、3D fullres、3D cascade四种配置选项
- 当用户选择配置时，系统应当显示该配置的适用场景和计算资源需求
- 当用户选择不合适的配置时，系统应当给出建议

#### FR-TRAIN-002 训练流程执行
**描述**: 系统应当能够执行完整的nnU-Net训练流程

**验收标准**:
- 当用户启动训练时，系统应当按照nnU-Net的标准流程执行预处理、网络架构搜索、训练和后处理
- 当训练过程中发生中断时，系统应当支持从检查点恢复训练
- 当训练完成后，系统应当自动保存训练好的模型权重和配置

#### FR-TRAIN-003 训练进度监控
**描述**: 系统应当提供训练进度的实时监控

**验收标准**:
- 当训练进行时，系统应当实时显示当前epoch、损失值、验证指标等信息
- 当训练进行时，系统应当提供训练曲线的可视化展示
- 当训练完成时，系统应当生成训练报告，包含最终的评估指标

#### FR-TRAIN-004 超参数配置
**描述**: 系统应当允许用户配置训练超参数

**验收标准**:
- 当用户配置超参数时，系统应当支持batch size、学习率、训练轮数等参数的设置
- 当用户设置不合理的超参数时，系统应当给出警告提示
- 当用户未设置超参数时，系统应当使用nnU-Net的默认值

#### FR-TRAIN-005 交叉验证
**描述**: 系统应当支持交叉验证训练

**验收标准**:
- 当用户选择交叉验证时，系统应当执行5折交叉验证
- 当交叉验证完成后，系统应当计算平均性能指标和标准差
- 当交叉验证完成后，系统应当保存每个fold的模型权重

### 3.4 模型推理模块

#### FR-INFER-001 单个影像推理
**描述**: 系统应当支持对单个CT影像进行肺叶分割推理

**验收标准**:
- 当用户提供单个CT影像时，系统应当在5分钟内完成分割推理
- 当推理完成时，系统应当输出五个肺叶的分割掩膜
- 当推理过程中发生错误时，系统应当记录错误日志并提示用户

#### FR-INFER-002 批量推理
**描述**: 系统应当支持批量CT影像的自动分割

**验收标准**:
- 当用户提供多个CT影像时，系统应当依次对每个影像进行推理
- 当批量推理执行时，系统应当显示当前处理的进度和预计剩余时间
- 当批量推理完成时，系统应当生成处理报告，包含所有影像的分割结果

#### FR-INFER-003 推理参数配置
**描述**: 系统应当允许用户配置推理参数

**验收标准**:
- 当用户配置推理参数时，系统应当支持选择使用哪个训练好的模型
- 当用户配置推理参数时，系统应当支持是否应用后处理的选项
- 当用户配置推理参数时，系统应当支持设置推理的batch size

#### FR-INFER-004 结果后处理
**描述**: 系统应当对推理结果进行后处理优化

**验收标准**:
- 当推理结果包含孤立的小区域时，系统应当应用形态学操作去除噪声
- 当推理结果的分割边界不连续时，系统应当应用平滑处理
- 当后处理完成后，系统应当确保每个肺叶区域的连通性

### 3.5 可视化模块

#### FR-VIS-001 2D切片可视化
**描述**: 系统应当提供CT影像和分割结果的2D切片可视化

**验收标准**:
- 当用户查看影像时，系统应当显示原始CT影像、分割掩膜和叠加结果
- 当用户浏览切片时，系统应当支持鼠标滚轮或滑动条切换切片
- 当用户调整窗口窗宽窗位时，系统应当实时更新显示效果

#### FR-VIS-002 3D体积渲染
**描述**: 系统应当提供肺叶分割结果的3D体积渲染

**验收标准**:
- 当用户查看3D渲染时，系统应当显示五个肺叶的三维模型
- 当用户查看3D渲染时，系统应当支持旋转、缩放、平移等交互操作
- 当用户查看3D渲染时，系统应当支持单独显示或组合显示不同肺叶

#### FR-VIS-003 分割结果叠加
**描述**: 系统应当将分割结果以半透明方式叠加在原始CT影像上

**验收标准**:
- 当用户查看叠加结果时，系统应当用不同的颜色标识不同的肺叶
- 当用户调整叠加透明度时，系统应当实时更新显示效果
- 当用户查看叠加结果时，系统应当提供图例说明颜色与肺叶的对应关系

#### FR-VIS-004 结果对比
**描述**: 系统应当支持分割结果与金标准的对比显示

**验收标准**:
- 当用户提供金标准标签时，系统应当同时显示预测结果和金标准
- 当进行结果对比时，系统应当高亮显示预测错误区域
- 当进行结果对比时，系统应当计算并显示差异统计信息

### 3.6 评估模块

#### FR-EVAL-001 性能指标计算
**描述**: 系统应当计算分割结果的性能指标

**验收标准**:

- 当计算完成时，系统应当生成详细的评估报告

**验收标准**:

- 当显示评估结果时，系统应当支持导出为图片或Excel格式

#### FR-EVAL-003 模型对比
**描述**: 系统应当支持不同模型版本的性能对比

**验收标准**:
- 当用户选择多个模型版本时，系统应当对比它们的性能指标
- 当进行模型对比时，系统应当显示各模型的优缺点
- 当进行模型对比时，系统应当给出模型选择的建议

#### FR-EVAL-004 错误分析
**描述**: 系统应当提供分割错误的详细分析

**验收标准**:
- 当分析错误时，系统应当统计各类错误的发生频率
- 当分析错误时，系统应当展示典型错误的案例
- 当分析错误时，系统应当给出可能的改进建议

## 4. 非功能需求

### 4.1 性能需求

#### NFR-PERF-001 推理速度
**描述**: 系统应当在合理时间内完成单次推理

**验收标准**:
- 当对标准分辨率的胸部CT影像（512x512x100）进行推理时，系统应当在5分钟内完成
- 当使用GPU加速时，推理时间应当在2分钟以内
- 当推理时间超过预期时，系统应当提示用户可能的原因

#### NFR-PERF-002 训练效率
**描述**: 系统应当在合理时间内完成模型训练

**验收标准**:
- 当使用nnU-Net 3D fullres配置训练时，系统应当在48小时内完成1000 epoch的训练
- 当训练过程中，系统应当合理利用GPU资源，GPU利用率应当保持在80%以上
- 当训练可以中断恢复时，系统应当定期保存检查点

#### NFR-PERF-003 内存占用
**描述**: 系统应当合理控制内存使用

**验收标准**:
- 当系统运行时，内存占用应当不超过可用内存的80%

| 术语 | 定义 |
|------|------|

| Hausdorff距离 | 衡量两个集合之间最大距离的指标，用于评估分割边界的准确性 |
| IoU | Intersection over Union，交并比，另一种分割评估指标 |
